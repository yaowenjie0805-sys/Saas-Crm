# Aster CRM 国内一流化（双速推进）设计说明

## 目标与范围

- 目标：将当前 CRM 打造成国内一流产品能力，覆盖中小企业、中大型企业与集团型企业。
- 战略：采用“双速推进”
  - 主线：销售主链（线索到回款）做深做透，快速形成业绩增量。
  - 副线：AI 自动化与国产生态并行建设，形成差异化与护城河。
- 北极星 KPI：赢单率提升（第一优先）；并行硬指标为人均产能提升、预测准确率/流程合规率提升。

## 分期规划

### 0-90 天（成交引擎 V1）

- 打通线索 -> 商机 -> 报价 -> 订单 -> 合同 -> 回款主链关键断点。
- 建立阶段门禁、折扣审批、逾期预警、核心漏斗看板。
- AI 侧优先落地：跟进摘要、评论回复、营销邮件、销售建议（支持手动模型 ID + Base URL + API Key）。
- 产出：可复制的成交流程模板 + 核心运营指标基线。

### 90-180 天（行业化与分层版本）

- 面向中小/中大/集团做分层能力包（流程复杂度、权限粒度、组织模型）。
- 推出 1-2 个行业模板（例如 ToB SaaS、制造）。
- 强化自动化规则中心与审批 SLA 治理。

### 180-360 天（平台化与护城河）

- 完成国产化生态深度连接与私有化部署标准方案。
- 完成统一模型网关、连接器市场化、模板市场化。
- 建成运营控制平面（增长、效率、合规一体化）。

## 产品架构蓝图

### 一、核心业务域（主线）

1. GTM 引擎
- 线索评分、分配、培育、转化。
- 基于来源质量、互动行为、组织画像进行优先级排序。

2. 交易引擎
- 商机阶段门禁（Stage Gate）
- 报价版本管理与折扣审批。
- 商机推进过程中的风险提示与下一步建议。

3. 履约回款引擎
- 合同里程碑与回款计划。
- 逾期预警、回款责任归因、复盘闭环。

4. 客户经营引擎
- 客户 360、组织关系、关键联系人地图。
- 续约/增购机会池与触达策略。

### 二、AI 自动化域（副线 A）

1. 销售 Copilot
- 跟进摘要、评论回复、营销邮件、销售建议。
- 所有能力统一支持模型 ID、Base URL、API Key 可配置。

2. 预测与运营 AI
- 赢单概率、管道健康评分、回款预测、异常解释。

3. 规则联动自动化
- 触发条件（金额阈值、超期、阶段停滞）
- 自动动作（创建任务、提醒、审批、升级）。

### 三、生态与国产化域（副线 B）

1. 连接器层
- 企微、钉钉、飞书、邮件、短信、呼叫、ERP/财务。

2. 模型网关层
- OpenAI、国产模型、本地模型统一接入。
- 标准化鉴权、超时、重试、审计与成本统计。

3. 部署与合规
- 多租户隔离、字段脱敏、审计追踪。
- 私有化部署、信创兼容、数据主权可控。

### 四、控制平面（平台能力）

- 租户与权限：RBAC + 数据范围控制。
- 流程与规则中心：审批、自动化、SLA。
- 指标与运营中心：漏斗、预测、合规、效率。
- 模板/插件机制：行业包、自动化包、连接器包。

## 关键数据流设计

1. 线索到赢单流
- Lead 创建/导入 -> 评分与分配 -> 跟进活动沉淀 -> 转商机 -> 报价审批 -> 订单合同 -> 回款。

2. AI 调用流
- 前端上下文收集 -> 模型网关（modelId/baseUrl/apiKey）-> 统一返回（code/requestId）-> 页面可追踪提示。

3. 自动化执行流
- 事件触发 -> 规则判断 -> 自动动作（任务/通知/审批）-> 结果审计。

4. 生态同步流
- 主数据变更 -> 事件/Outbox -> Connector 下发 -> 回执落库与失败补偿。

## 错误处理与可靠性

- 统一错误结构：errorCode、message、requestId、operation、retriable。
- 分级降级：AI 失败时可降级到模板结果；连接器失败进入重试队列。
- 幂等与补偿：关键事务使用幂等键与状态机防重复。
- 可观测性：按租户/模块/模型维度监控成功率、延迟、失败原因。

## 测试与验收体系

### 测试分层

- 单元测试：领域规则（评分、门禁、审批、预测）
- 集成测试：主链路、AI 网关、连接器
- E2E 测试：销售、经理、财务、管理员关键场景

### 发布门禁

- 主链冒烟必须通过。
- 关键指标阈值不回退（成功率、延迟、自动化成功率）。

### 90 天验收指标

- 赢单率：基线对照显著提升（目标建议 +8%）
- 人均产能：有效跟进与推进效率提升（目标建议 +30%）
- 预测/合规：预测准确率与流程合规率提升（目标建议 85%+）
- 稳定性：核心接口可用性与 P95 延迟达标。

## 风险与治理

- 风险 1：范围过大导致交付失焦
  - 对策：双速推进下保持主线优先级，副线能力包按里程碑上线。

- 风险 2：AI 成本和效果波动
  - 对策：模型网关分层路由、降级策略、成本看板与预算告警。

- 风险 3：多生态接入复杂度高
  - 对策：统一连接器协议、回执审计、灰度发布与回滚手册。

## 结论

该方案以“主线拿结果、副线建护城河”为原则，兼顾短期业务增长与长期平台化。按本设计推进，可在 90 天内形成可验证的增长与效率改进，在 360 天内达到国内一流 CRM 的产品与工程成熟度。
