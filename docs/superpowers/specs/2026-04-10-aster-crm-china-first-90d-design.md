# Aster CRM 中国一流化设计（90天冲刺版）

## 1. 目标与边界

### 北极星目标
- 在现有项目基础上，90天内将 CRM 打造成“可规模复制的成交引擎”。
- 以成交效率与经营可观测性为第一目标，AI 作为成交增益器而非孤立功能。

### 本阶段优先级
- 执行顺序：`1 > 3 > 2`
- 含义：先业务主链（1），再平台与生态底座（3），最后 AI 放大（2）。

### 本阶段范围（In Scope）
- 成交主链标准化：`Lead -> Opportunity -> Quote -> Order -> Contract -> Payment`
- 阶段门禁与审批一致性：关键状态强约束、可解释错误码
- 连接器可靠性：重试/限流/审计/追踪闭环
- KPI 看板：`winRate`、`forecastAccuracy`、`pipelineHealth`、`taskDoneRate`
- AI 四入口稳定化：跟进摘要、评论回复、营销邮件、销售建议（支持手动模型接入）

### 非范围（Out of Scope）
- 大规模前端框架重构
- 行业模板全覆盖
- 重 BI 平台替代（本阶段仅交付经营级看板）

## 2. 总体架构（1 > 3 > 2）

### 业务层（优先级1）
- 构建主链状态机与门禁规则，禁止关键环节跳步。
- 所有关键动作统一带 `requestId/operator/tenant/timestamp`。
- 订单、合同、回款形成可追踪闭环。

### 平台层（优先级3）
- 合规治理：审批流、审计日志、权限边界、租户隔离。
- 集成底座：Webhook/连接器统一策略（重试、熔断、审计）。
- 可观测：接口质量与业务指标双维度监控。

### AI层（优先级2）
- 统一 AI 契约：`modelId + baseUrl + apiKey` 全链路透传。
- AI 与业务动作绑定，失败可降级且不阻断成交流程。

### 前后端协同原则
- 后端 `summary` 为指标真源，前端只做展示与格式化。
- 错误码集中维护，前后端一一映射。
- `localizedMetrics` 仅补充，不覆盖主口径。

## 3. 90天里程碑与验收

### D1-D30：打穿主链
- 交付：阶段门禁生效、审批一致性校验、关键错误码统一。
- 验收：
- 关键流程成功率 > 98%
- 非法状态跳转下降 80%+
- 关键链路时延下降 20%+

### D31-D60：平台稳态
- 交付：连接器可靠投递、审计字段闭环、基础经营看板上线。
- 验收：
- Webhook 投递成功率（含重试）> 99%
- 回调失败可追踪率 100%
- P1/P2 故障恢复时间明显下降

### D61-D90：AI放大成交
- 交付：4 个 AI 入口稳定上线、KPI 看板补齐、建议到执行闭环。
- 验收：
- 销售人均有效触达提升
- 赢单率/预测准确率可量化提升
- AI 功能可用率 > 99%，且失败可降级

## 4. 技术实现细则

### 数据口径
- KPI 统一后端输出：`winRate/forecastAccuracy/pipelineHealth/taskDoneRate`。
- 指标字段必须覆盖空值边界与舍入规则测试。

### 错误码规范
- 错误码必须机器可读：`code + details.requiredStatus`。
- 关键码变更保留兼容窗口，前端映射需同时覆盖新旧码。

### 测试策略
- 后端：控制器契约测试 + 服务单测 + 主链集成测试。
- 前端：API payload、组件交互、错误码映射、KPI 展示一致性测试。
- 发布前必跑：主链、连接器、AI 四入口、KPI 看板、web build。

### 发布策略
- 小步可回滚；每次发布必须附带验证命令与回滚点。
- 生产监控必须包含：异常率、时延、错误码分布、重试率。

## 5. 风险与兜底

### 风险1：错误码漂移
- 兜底：前后端错误码映射回归测试纳入必跑。

### 风险2：KPI 口径冲突
- 兜底：后端 `summary` 单一真源；前端禁止重算替代主口径。

### 风险3：连接器抖动
- 兜底：`retryable + requestId + dispatched` 全链路审计。

### 风险4：AI 可用性波动
- 兜底：统一降级策略，AI 不可用时不阻断成交链路。

## 6. 多agent执行方式

- Agent A：成交主链与状态门禁
- Agent B：连接器与通知可靠性
- Agent C：AI 契约与入口能力
- Agent D：指标口径与看板展示
- 主控：统一回归、冲突处理、质量闸门、发布编排

## 7. 完成定义（DoD）

- 需求：覆盖本设计全部 In Scope 项
- 测试：后端关键套件 + 前端关键套件 + web build 全绿
- 质量：无阻塞级审查问题
- 交付：可追溯 commit、可回滚、可监控
